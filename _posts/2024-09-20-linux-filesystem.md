## Иерархия файловой системы

- `/` : Корневой каталог
  - `/bin` (бинарные файлы): Основные утилиты пользователя (ls, cat etc..).
  - `/boot`: Загрузочные файлы.
  - `/dev`: Файлы устройств.
  - `/etc` Общесистемные конфигурационные файлы.
  - `/home`: Домашние каталоги пользователей.
  - `/lib`, `/lib32`, `/lib64`: Основные библиотеки, программное обеспечение (статическая часть).
  - `/media`: Точки монтирования для сменных носителей.
  - `/mnt`: Для временного монтирования файловой системы.
  - `/opt`: Дополнительное программное обеспечение.
  - `/proc`: Псевдо-файловая система - информация об ядре, процессах.
  - `/root`: Домашний каталог пользователя root.
  - `/run`: Данные необходимые для работы демонов.
  - `/sbin`: Основные команды для администрирования.
  - `/snap`: snap пакеты (для Ubuntu).
  - `/srv`: Данные для служб.
  - `/sys` : Псевдо-файловая система. Содержит информацию об устройствах.
  - `/tmp`: Временные файлы.
  - `/usr`: Вторичная иерархия программного обеспечения.
    - `/usr/bin`: Дополнительные программы для всх пользователей.
    - `/usr/include`: Заголовочные данные.
    - `/usr/lib`: Дополнительные библиотеки, программное обеспечение (статическая часть).
    - `/usr/local`: Третичная иерархия программного обеспечения.
    - `/usr/sbin`: Дополнительные системные программы администратора.
    - `/usr/share`: Архитектурно - независимые данные.
    - `/usr/src`: Исходные кодв прграммного обеспечения.
  - `/var`: Изменяемые файлы.
    - `/var/lib`: Данные, изменяемые программами в процессе работы.
    - `/var/lock`: Файлы блокировки.
    - `/var/log`: Журналы.
    - `/var/mail`: Почтовые ящики пользователей.
    - `/var/spool`: Очереди.

## Навигация по файловой системе

Команда                                     | Описание
--------------------------------------------|------------------------------------------------------------------------------------------------
`cd <имя_каталога>`                         | переход в каталог
`cd ~ <имя_пользователя>`                   | перейти в домашний каталог указанного пользователя
`cd -`                                      | вернуться в предыдущий каталог
`pwd`                                       | вернуть абсолютное имя текщего каталога
`ls`                                        | получение информации о файлах/каталогах (основные параметры те же, что и у `dir`)
`ls -l`                                     | подробная информация о файлах и каталогах
`ls -d`                                    | информция про каталог, а не о его содержимом, как правило используется вместе с параметром `-i`
`ls -a`                                     | информация о всех файлах, включая скрытые, имена которых с `.`
`ls -i`                                    | выводит номера индексных дискрипторов.
`tree`                                      | вывод информации о содержимом каталогов в древовидном формате

## Команды создания файлов

Команда                                     | Описание
--------------------------------------------|------------------------------------------------------------------------------------------------
`touch <имя_файла>`[^footnote]              | создание пустрого регулярного файла
`mkdir <имя_каталога> (-p)`                 | создание нового каталога. Параметр `-p` создает ветку пустых каталогов
`ln -s <имя_файла> <имя_ссылки>`[^fn-nth-2] | создание символической ссылки (ярлык)
`ln <имя_файла> <имя_жесткой_ссылки>`       | создание жесткой ссылки
`mknod`                                     | создание специального файла устройства
`mkinfo`                                    | создание именованного канала

## Операции с файлами

Команда             | Описание
--------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
cp (-a, -r, -s, -d) | Копирование файлов и каталогов
cp -a               | (ранее -rpd). Копируется ветка каталогов с сохранением основных атрибутов файлов и каталогов (владелец, группа-владелец, права доступа). Символическая ссылка - остается ссылкой.
cp -r               | рекурсивное копирование (копирование ветки каталогов со всем содержимым)
cp -s               | создается символическая ссылка вместо копирования файла
cp -d               | если копируется символическая ссылка, то создается символическая ссылка, а не файл
rm (-r, -f)         | удаление файлов
rm -r               | рекурсивное удаление, т.е. удаление ветки каталогов со всем содержимым
rm -f               | подтверждение удаления файлов и каталогов не требуется

## Другие полезные утилиты по работе с файлами

Команда                                     | Описание
--------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------
rmdir <имя_каталога>[^fn-nth-3]             | удаление пустрого каталога
file                                        | информация о характере содержимого файла
which <имя_программы>                       | показывает расположение основного файла программы, т.е. осуществляет поиск программы по имени в каталогах, перечисленных в глобальной переменной PATH
find <точка_начала_поиска> <параметр_поиса> | поиск файлов по его атрибутам ([**подробней**]())

---
[^footnote]: Создается файл с заданным именем если такого файла не существовало, или обновляет времена доступа и изменения на текущее, если файл уже был создан ранее.
[^fn-nth-2]: Альтернативный способ создания символической ссылки: `cp -s <имя_файла> <имя_ссылки>`
[^fn-nth-3]: Пустой каталог - файл, содержащий две записи для имен каталогов. `.`(точка) и `..`(две точки): `ls -a <имя_пустого_каталога>`.
