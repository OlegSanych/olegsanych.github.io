Для настройки SSH авторизации по ключу `ed25519` в Debian Linux выполните следующие шаги:

### Шаг 1: Создание пары ключей `ed25519`

1. Откройте терминал на локальной машине (той, с которой будет выполняться подключение).
2. Выполните команду для создания новой пары ключей `ed25519`:

   ```bash
   ssh-keygen -t ed25519 -C "Комментарий_к_ключу"
   ```

3. Когда появится запрос на указание пути сохранения ключа, по умолчанию можно использовать `~/.ssh/id_ed25519` или указать другое место. Нажмите `Enter`, чтобы принять путь по умолчанию.
4. При запросе пароля (фразы-пароля) можно либо задать его (для дополнительной защиты), либо оставить пустым и нажать `Enter`, чтобы пропустить.

В результате будут созданы два файла:

- Приватный ключ: `id_ed25519`
- Публичный ключ: `id_ed25519.pub`

### Шаг 2: Копирование публичного ключа на сервер

1. Скопируйте публичный ключ на удалённый сервер. Это можно сделать командой `ssh-copy-id`, указав имя пользователя и адрес сервера:

   ```bash
   ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server_ip
   ```

   *Замените `user@server_ip` на ваш логин и IP-адрес сервера.*

2. Если команда `ssh-copy-id` недоступна, выполните следующие шаги:
   - Откройте публичный ключ в текстовом редакторе:

     ```bash
     cat ~/.ssh/id_ed25519.pub
     ```

   - Скопируйте содержимое ключа.
   - На сервере откройте файл `~/.ssh/authorized_keys` для редактирования (создайте его, если он отсутствует):

     ```bash
     nano ~/.ssh/authorized_keys
     ```

   - Вставьте содержимое публичного ключа в новую строку и сохраните файл.
   - Убедитесь, что права доступа к файлу настроены корректно:

     ```bash
     chmod 600 ~/.ssh/authorized_keys
     ```

### Шаг 3: Настройка прав доступа на сервере

1. Проверьте, что у директории `~/.ssh` на сервере правильные права:

   ```bash
   chmod 700 ~/.ssh
   ```

2. Убедитесь, что права на файл `authorized_keys` установлены на `600`, как указано выше.

### Шаг 4: Настройка SSH-доступа по ключу на сервере

1. На сервере откройте файл конфигурации SSH для редактирования:

   ```bash
   sudo nano /etc/ssh/sshd_config
   ```

2. Убедитесь, что в файле установлены следующие параметры:

   ```bash
   PubkeyAuthentication yes
   PasswordAuthentication no  # Отключает вход по паролю (опционально)
   ```

   *Примечание:* Отключение входа по паролю (`PasswordAuthentication no`) повысит безопасность, но убедитесь, что авторизация по ключу настроена корректно.

3. Перезапустите SSH-сервер, чтобы применить изменения:

   ```bash
   sudo systemctl restart ssh
   ```

### Шаг 5: Подключение к серверу с использованием ключа

1. На локальной машине выполните команду для подключения к серверу:

   ```bash
   ssh -i ~/.ssh/id_ed25519 user@server_ip
   ```

   *Замените `user@server_ip` на логин и IP-адрес сервера.*

2. Если всё настроено правильно, соединение установится без запроса пароля.

### Шаг 6: Настройка автоматического использования ключа (опционально)

Чтобы не указывать ключ при каждом подключении:

1. Откройте файл конфигурации SSH на локальной машине:

   ```bash
   nano ~/.ssh/config
   ```

2. Добавьте запись:

   ```bash
   Host server_name
       HostName server_ip
       User user
       IdentityFile ~/.ssh/id_ed25519
   ```

   *Замените `server_name`, `server_ip` и `user` на имя сервера, его IP-адрес и ваше имя пользователя.*

Теперь для подключения можно просто использовать:

```bash
ssh server_name
```

На этом настройка авторизации по ключу `ed25519` завершена.
